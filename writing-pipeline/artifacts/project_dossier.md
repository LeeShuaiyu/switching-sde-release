## Project Objective

The project studies single-switch inference in stochastic dynamics. The scientific target is to infer the changepoint time `\tau` of a trajectory generated by one regime before the switch and another regime after the switch, while also preserving a posterior distribution over admissible switching indices instead of only producing a point estimate.

The legacy `PENN` repository began as a parameter-estimation network for alpha-stable Lévy-noise SDEs, but the switching subsystem extends it toward changepoint posterior inference for switching Ornstein-Uhlenbeck and switching Duffing systems. The release wrapper repackages those results into a self-contained report-and-paper workflow.

## Problem Setting

The task is a single-switch changepoint problem on a scalar trajectory `x_{1:L}`. The admissible changepoint set is interior to the sequence, with `switch_time_range = [0.2, 0.8]` and `split_min_seg_len = 16` in the main switching configuration.

The benchmark families are:

- switching Ornstein-Uhlenbeck trajectories for the linear suite
- switching Duffing trajectories for the nonlinear suite

The main evaluation families visible in the final benchmark reports are:

- `fixed_v2` for the linear ID benchmark
- `fixed_v4_s1`, `fixed_v4_s2`, `fixed_v4_s3` for linear stress tests
- `fixed_v5_nl_id` for nonlinear ID
- `fixed_v5_nl_s1`, `fixed_v5_nl_s2`, `fixed_v5_nl_s3` for nonlinear stress tests

The stress scenarios have clear semantic meanings rather than just shorthand labels:

- `S1` / `OOD_NL1`: heavy-tailed innovations plus sparse impulse corruption
- `S2` / `OOD_NL2`: low regime separability
- `S3` / `OOD_NL3`: irregular observation with missingness and burst dropouts

## Main Method

The main method family is a candidate-constrained posterior estimator for changepoint inference.

Core components:

- a sequence backbone, usually `LSTM` or `Transformer`
- a posterior head that scores admissible split points and normalizes them into a discrete posterior over `\tau`
- an optional auxiliary regime-ratio branch
- an optional deterministic regression head for comparison in the dual setup
- an optional segmentation head for a state-segmentation baseline

The split posterior implementation is in `../PENN/switching/models/heads.py` and `../PENN/switching/models/switching_estimator.py`. The split head builds left and right segment summaries, combines them into a feature vector consisting of the left summary, right summary, their difference, their elementwise product, and normalized time, then masks candidate times outside the admissible window. The posterior utilities in `../PENN/switching/posterior_utils.py` then turn those logits into categorical or hazard posteriors, MAP indices, top-k modes, and highest-posterior-density intervals.

The clean manuscript description is therefore not a raw repository name like `split_dual`, but a constrained dual-branch posterior head for switching-time inference.

## Final Results

The final result depends on the benchmark family, and the strongest method is not identical across all suites.

- In `reports/paper_full/`, the main table shows `Ours_transformer_hard` as the strongest listed deep method on the paper-grade linear benchmark, with `MAE_time = 1.6383`, `F1@0.02 = 0.3015`, and `collapse_rate = 0.0`. In the same table, `D2_transformer_regression` is worse at `2.1270 / 0.0815`, and `D4_transformer_segmentation` is worse at `1.8925 / 0.1247`; the classical baselines also trail on posterior quality.
- In the same `paper_full` evidence, the stability snapshot shows `Ours_lstm_hard` as much stronger on the fixed-v2 split, with `worst MAE = 0.7757`, `worst F1 = 0.6620`, and `collapse_rate = 0.0`.
- In `reports/p5/`, the ID benchmark table gives `Ours_lstm_A0` the best ID score: `MAE_time_mean = 0.6227`, `F1@0.02_mean = 0.6943`, `collapse_rate = 0.0`.
- In the same P5 suite, the mild stress `S1` is still handled well by `Ours_lstm_A0` at `0.8854 MAE` and `0.5395 F1`, while the stronger stress settings `S2` and `S3` degrade sharply and can collapse.
- In `reports/p6_nonlinear_full_v3/`, the nonlinear ID table gives `Ours_lstm_A0` `MAE_time_mean = 0.3484`, `F1@0.02_mean = 0.8367`, and `collapse_rate = 0.0`; the transformer variant is also strong at `0.4991 / 0.7573 / 0.0`.
- In the nonlinear stress suite, `OOD_NL1` remains a reliability warning: both posterior backbones keep strong point metrics but still show collapse, while `OOD_NL2` and `OOD_NL3` remain stable and competitive.

The main empirical message is that posterior changepoint modeling clearly beats simpler deep baselines on posterior quality and calibration, but robustness depends strongly on the shift mechanism and on which backbone is used.

## Model Architecture

The architecture is easiest to summarize as three layers:

1. a backbone encoder (`LSTM` or `Transformer`)
2. a split posterior head over admissible changepoints
3. auxiliary heads for ratio prediction, optional regression, or segmentation depending on the experiment

More specifically:

- the backbone produces latent sequence features of dimension `64`
- `SequenceSwitchHead` gives per-time-step logits for the sequence baseline
- `SplitPosteriorHead` constructs left and right segment summaries, then combines them with difference and elementwise-product features plus normalized time
- `RatioHead` predicts the occupancy ratio over the two regimes
- `RegressionSwitchHead` is used for deterministic regression comparisons
- `StateSegHead` supports the segmentation baseline

The split-posterior head optionally augments the learned features with segment statistics and a residual gate, but that path is treated as an ablation rather than the main result.

## Training and Inference Procedure

The training and inference pipeline is explicit in code and scripts:

- `prepare_fixed_split.py` creates the fixed ID split manifests
- `build_stress_splits.py` creates the linear stress manifests and applies observation shifts when needed
- `build_nonlinear_splits.py` and the nonlinear stress utilities do the same for the Duffing suite
- `run_oracle_baselines.py` precomputes plugin teacher probabilities and reliability statistics
- `run_teacher_overfit_check.py` sanity-checks whether the teacher is learnable before full benchmark runs
- `train_switching.py` trains the model with hard or distillation losses, optional curriculum, optional weighted sampling, and optional teacher gating
- `eval_switching.py` evaluates on eval/test splits and exports JSON, NPZ, calibration, top-k, HPDI, and reliability summaries

The inference path also includes temperature calibration on the validation split for posterior metrics.

The evaluation metrics include:

- `MAE_time`
- `F1@0.02`
- `NLL_cal`
- `ECE_cal`
- `topk_hit`
- `ci90_coverage`
- `ci90_width_mean_time`
- `rank_corr_to_teacher`
- `expected_abs_err_time`
- `candidate_auc@0.02`
- `collapse_flag`

The collapse detector is not a vague label; it is a rule-based check that trips when calibrated NLL, rank correlation to teacher, or top-k hit fall past configured thresholds.

## Challenges and Detours

- The project began as a broader parameter-estimation repository, so the switching subsystem had to be made legible in terms that are not the legacy repository's original framing.
- There are multiple historical result branches and folders, including `autodl`, `autodl_p2`, `paper_full`, `p5`, `p6_nonlinear_full_v3`, and smoke variants. The final manuscript should not blur these together.
- The strongest backbone is not uniform across all suites. The linear paper-full narrative is transformer-centered, but the fixed-v2 and nonlinear ID tables show stronger LSTM rows in several places.
- Distillation and segment-statistics variants were explored, but the plain hard posterior configuration is the best-supported final recipe.
- The linear stress suite shows a real failure mode under low separability and irregular observation.
- The nonlinear OOD_NL1 case has strong point metrics but still triggers collapse, so point accuracy alone would overstate reliability.

## Terminology Risks

- Raw labels such as `Ours_transformer_hard`, `Ours_lstm_A0`, `D1`, `D2`, `D3`, and `D4` are evidence labels, not prose labels.
- Raw implementation names such as `split_dual`, `teacher_probs`, `cp_idx`, and `rank_distill` should be translated into domain language or notation before writing the paper.
- Folder names such as `paper_full`, `p5`, `p6_nonlinear_full_v3`, and `autodl` are provenance labels and should not appear in polished narrative text unless the sentence is explicitly about artifact provenance.
- The manuscript should use terminology like posterior changepoint inference, candidate-constrained split posterior, regime occupancy, calibration, and collapse diagnostics instead of code-like project shorthand.

## Open Uncertainties

- The manuscript should not claim that one backbone is universally best. The linear paper-full table, the P5 fixed-v2 table, and the P6 nonlinear table do not support a single backbone claim across all suites.
- The exact reason the transformer-centered paper-full narrative diverges from the stronger LSTM rows in the fixed-v2 and nonlinear ID tables is not explained in the code or reports.
- The severe-shift failure modes in `S2`, `S3`, and `OOD_NL1` are empirically visible but not yet mechanistically explained.
- The legacy project does not ship a pinned environment file, so reproduction still depends on reconstructing the tested stack from documentation and code.
